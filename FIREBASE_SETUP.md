# Firebase Firestore Setup

## Overview
This project now uses Firebase Firestore to save B2B lead data directly to your database instead of making API calls.

## Setup Instructions

### 1. Environment Variables
Create a `.env.local` file in the project root with your Firebase configuration:

```env
VITE_FIREBASE_API_KEY=your_api_key_here
VITE_FIREBASE_AUTH_DOMAIN=your_project_id.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project_id.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

### 2. Get Firebase Configuration
1. Go to your Firebase Console (https://console.firebase.google.com/)
2. Select your project
3. Go to Project Settings (gear icon)
4. Scroll down to "Your apps" section
5. If you haven't added a web app, click "Add app" and select Web
6. Copy the configuration values to your `.env.local` file

### 3. Firestore Database Setup
1. In Firebase Console, go to "Firestore Database"
2. Make sure your database is created and rules allow writes
3. The app will create a `leads` collection automatically when the first lead is saved

### 4. Firestore Security Rules (Optional)
For production, consider updating your Firestore rules:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /leads/{document} {
      allow read, write: if true; // Adjust based on your security needs
    }
  }
}
```

## Usage
The `saveLeadData` function in `useSaveLeadData` hook now automatically saves to Firestore. No changes needed in your React components.

## Data Structure
Lead data is saved to the `leads` collection with the following structure:
- All B2bLead fields (Fname, Lname, Tel, Email, Company, CompanyID, InterestedProject, Source, PDPA, TypeInterest)
- `createdAt`: Server timestamp
- `updatedAt`: Server timestamp
- Document ID: Auto-generated by Firestore

## Troubleshooting
- Make sure all environment variables are set correctly
- Check Firebase Console for any permission issues
- Verify your Firestore database is in the correct region
- Check browser console for any Firebase-related errors
